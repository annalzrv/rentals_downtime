# Детальный план задач по ролям

#### 1. Архитектор (Анна)

Оценка времени: 12–14 часов

- **1 час Старт:** Написать класс `State` (TypedDict) с полями `next_node`, `mse_history`, опционально `supervisor_reasoning`; закинуть в репозиторий. Это разблокирует работу всей команды.  
- **2 часа Инфраструктура LLM:** Поднять локальный инференс (Ollama или vLLM) для Qwen2.5-Coder и Llama-3/Mistral. Написать базовые обертки для вызова моделей.  
- **3 часа Скелет LangGraph:** Создать узлы (nodes) и прописать логику переходов (conditional edges). После `**Code_Executor`**: при ошибке → `**Coder_Agent**`, при успехе → `**Supervisor_Agent**`. После Supervisor — единая функция `**route_after_supervisor(state)**` с guardrails (не размазывать `if MSE` по разным местам). На первом этапе узлы могут передавать mock data.  
- **4 часа Промпт-инжиниринг:** Строгие System Prompts в трёх блоках: (1) `**RAG_Domain_Expert`** — список идей; (2) `**Coder_Agent**` — JSON с полями `explanation` и `code` (без маркдауна вокруг JSON); (3) `**Supervisor_Agent**` — JSON `reasoning` + `next_node` (`RAG_Domain_Expert` | `Coder_Agent` | `END`).  
- **4 часа Сборка (Интеграция):** Объединить модули участников в единый пайплайн. Дебаг бесконечных циклов; **тесты guardrails**: принудительный `END` при достижении max iter, корректный fallback при неверном `next_node` или битом JSON.

#### 2. RAG-Инженер

Оценка времени: 10–12 часов

- **2 часа Сбор данных:** Найти 3–5 качественных материалов. Обязательно: ноутбук-бейзлайн с Kaggle по аренде, статья по feature engineering для временных рядов (Time Series), мануал по CatBoost.  
- **2 часа Векторная БД:** Поднять ChromaDB (или FAISS), настроить чанкинг (разбивку текста на куски) и эмбеддинги (можно использовать легковесные модели типа `all-MiniLM-L6-v2`).  
- **3 часа Поиск и Retrieval:** Написать Python-функцию, которая принимает на вход описание датасета (от EDA-узла) и возвращает топ-3 релевантных куска текста из базы.  
- **3 часа Интеграция с промптом:** Совместно с Архитектором протестировать, как `Llama-3` реагирует на подкинутый контекст. Добиться, чтобы модель генерировала адекватное текстовое ТЗ на фичи.  
- *(Опционально)* Учесть, что при возврате в RAG из цикла `features_plan` может **дополняться** — при необходимости согласовать отдельное поле в State позже.

#### 3. DevOps / Security

Оценка времени: 12–14 часов

- **3 часа Изоляция (Песочница):** Написать функцию-экзекутор. Она должна сохранять сгенерированный LLM код во временный файл `agent_script.py` и запускать его через `subprocess.run()`. Настроить жесткий timeout (например, 300 секунд на обучение).  
- **2 часа Парсинг логов:** Написать перехват `stderr` (ошибок) и `stdout`. Если код падает с ошибкой синтаксиса или `KeyError`, функция должна возвращать чистый `Traceback` в граф, чтобы кодер мог исправиться. Итоговые логи в `execution_result` должны быть доступны **Supervisor** в state.  
- **3 часа Data Validation (Проверка сабмита):** Написать скрипт, который проверяет итоговый `submission.csv` перед отправкой на Kaggle (длина файла совпадает с `test.csv`, нет NaN, значения в нужных числовых диапазонах).  
- **4 часа Docker-контейнеризация:** Упаковать систему (или хотя бы узел экзекутора) в Docker, чтобы гарантировать безопасность хост-машины.

#### 4. MLOps / Метрики (Боря)

Оценка времени: 10–12 часов

- **1 час Базовый EDA-скрипт:** Написать тот самый стартовый узел `Data_Profiler`, который делает `.info()`, считает статистику по таргету и отдает текст.  
- **3 часа Кросс-валидация:** Написать эталонный код для `Time Series Split` и подсчета MSE. Писать текущий MSE в `metrics` и **append в `mse_history`** после каждой успешной валидной итерации (как договоритесь в пайплайне). *Важно:* эталонный код вставить в системный промпт `Coder_Agent` как пример, чтобы он не делал глупую `train_test_split` на данных, зависящих от времени. Порог `target_threshold` держать в конфиге и передавать в промпт Supervisor (совместно с Архитектором).  
- **3 часа Телеметрия (Бенчмаркинг):** Написать декораторы, которые оборачивают вызовы LLM и логируют: время генерации, потраченные токены, количество итераций графа до успешного завершения. Сохранять это в `experiment_log.json`.  
- **4 часа Документация (README):** Нарисовать схему графа (можно через draw.io или экспорт из LangGraph), описать процесс запуска системы для преподавателей, оформить репозиторий. В README описать **agentic routing (Supervisor)** и **guardrails** в Python (`route_after_supervisor`).
